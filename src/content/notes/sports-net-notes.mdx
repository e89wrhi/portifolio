# ⚽ Sports-Net: Technical Notes

## What
A comprehensive sports platform providing real-time match data, AI-driven outcome predictions, and a community voting system.

## Why
The goal was to create an interactive sports experience that goes beyond simple live scores by providing data-driven insights and engaging users through weighted sentiment analysis.

## How
- **Real-time Pipeline**: Uses .NET Core on the backend to push match updates to the frontend via WebSockets/SSE.
- **AI Engine**: Employs historical data analysis models via Microsoft Extensions AI to generate "Win/Draw/Loss" probabilities.
- **Data Fetching**: Optimized with **TanStack React Query** to handle thousands of concurrent state updates without flickering.
- **Community Layer**: A voting system where "weight" is determined by user engagement and accuracy history.

## Challenges & Solutions
- **Challenge**: Managing the high velocity of data during live match events.
- **Solution**: Implemented a debouncing and batching strategy for state updates to ensure the UI remains responsive even during peak traffic.

## Next Steps
- Integration with live sports betting APIs for odd-comparison features.
- Expansion into other sports beyond football (Basketball, Tennis).
